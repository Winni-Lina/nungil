# 눈길 (Nungil)

> 지적장애인과 보호자를 연결하는 AI 기반 일상 지원 앱

지적장애인이 일상 과업을 스스로 수행할 수 있도록 AI 어시스턴트 **똘똘이**가 단계별로 안내하고, 보호자가 원격으로 일정을 관리·모니터링할 수 있는 안드로이드 앱입니다.

## 서비스 개요

| 구분 | 내용 |
|------|------|
| 대상 | 지적장애인(사용자) + 보호자 |
| 연동 방식 | QR 코드 1대1 페어링 |
| AI 어시스턴트 | 똘똘이 (Gemini API 기반) |
| 음성 입력 | Google Cloud STT |
| 알림 | Firebase Cloud Messaging (FCM) |

## E2E 시나리오

### 시나리오 1 — 서비스 시작 및 연동

```
[사용자]  앱 설치 → QR 코드 생성
    ↕  QR 스캔으로 즉각 기기 연동
[보호자]  앱 설치 → QR 코드 스캔
```

| 단계 | 내용 |
|------|------|
| **① 서비스 시작 및 연동** | 사용자가 QR 생성, 보호자가 QR 스캔 → 즉각 기기 연동 |
| **② 가이드** | 앱 실행 시 똘똘이 등장, 보호자·사용자 각각 맞춤형 튜토리얼 진행 |
| **③ 화이트리스트 등록** | 보호자가 사용자가 안전하게 수행 가능한 과업 목록 등록 |

---

### 시나리오 2 — 일정 등록 및 수행

| 단계 | 내용 |
|------|------|
| **④ 일정 등록** | 보호자가 일정 등록 (예: 4월 11일 오전 10시, 세탁실에서 가루세제로 빨래하기) |
| **⑤ 일정 수행** | 정해진 시간에 사용자에게 알림 발송 → 똘똘이가 단계별 지침 안내 |

**수행 중 인터랙션**

| 버튼 | 동작 |
|------|------|
| ✅ 다했어 | 다음 단계로 진행 |
| ❓ 궁금해요 | 사진 촬영 후 질문 (예: "섬유유연제가 뭐야?") → 똘똘이 분석 후 답변 |
| 집중력 유지 | 과업과 무관한 질문 시 "지금은 빨래에 집중해요"로 부드럽게 차단 |

---

### 시나리오 3 — 자유 시간 질문

```
사용자: 똘똘이 아이콘 클릭 or "똘똘아~" 호출
똘똘이: "무엇이 궁금한가요?" + 예상 질문 3개 제시
```

| 질문 유형 | 처리 방식 |
|-----------|-----------|
| **일반 질문** | 이해하기 쉬운 말로 답변 |
| **과업 관련 질문** | 화이트리스트 등록 과업이면 일정 미등록 상태여도 단계별로 안내 |

---

### 시나리오 4 — 보호자 알림 및 보고서

**보호자 알림 발송 조건**

| 케이스 | 조건 | 동작 |
|--------|------|------|
| 일정 미수행 | 알림 후 미응답 (2차 리마인드) | 보호자에게 알림 발송 |
| 반복 질문 후 이해 불가 | 궁금해요 3회 설명 후에도 모를 경우 | 보호자에게 알림 발송 |

**활동 보고서** (모든 일정 완료 후 보호자 앱에 자동 생성)

1. 일정 달성도
2. 수행 중 질문한 항목
3. 가이드만으로 완료한 일정
4. 보호자 알림이 발생한 일정

## 유스케이스 다이어그램

### 사용자 (장애인)

```
사용자 (장애인)
 ├── 질문하기
 │    ├── 카메라 촬영
 │    ├── 음성 인식
 │    └── 질문 버튼 선택
 │         └── [데이터 전달]
 │              → 위험물 및 특이사항 분석
 │              → 결과 생성
 │              → 맞춤형 답변 및 가이드 수신
 └── 일정 시스템
      ├── 일정 알림 발생 → (자동 전환) → 단계별 일정 안내 수행
      │                                    ├── 완료 여부
      │                                    └── 일정 미수행 시 → 데이터 제공
      └── 초기 설정 및 연동
           ├── 사용자 1대1 연결
           └── 역할 확정 및 화면 분기
```

### 보호자

```
보호자
 ├── 일정 관리 (등록 / 수정 / 삭제)
 ├── 일정 시행 여부 확인
 ├── 미수행 일정 알림 수신
 ├── 위험물 및 특이사항 입력
 └── 질문 내역 확인
```

## 실행 방법

### 서버

1. Oracle DB 실행 후 스키마 적용
2. `server/src/main/resources/application.properties` 생성 (아래 형식 참고)

```properties
google.ai.api-key=YOUR_GEMINI_API_KEY
db.driver=oracle.jdbc.OracleDriver
db.url=jdbc:oracle:thin:@localhost:1521/orcl
db.username=YOUR_DB_USERNAME
db.password=YOUR_DB_PASSWORD
```

3. 환경변수 설정

```
GOOGLE_APPLICATION_CREDENTIALS=C:\path\to\stt-service-account.json
```

4. Eclipse에서 Tomcat 서버 실행

### 클라이언트

1. Android Studio에서 `client/` 프로젝트 열기
2. `google-services.json` 배치 (`client/app/` 하위)
3. 기기 또는 에뮬레이터에서 실행
