# 시연 비상 영상 가이드

발표 중 라이브 시연이 실패할 경우를 대비해 미리 녹화해둘 영상 목록.
공통 권장 사양: **1080p, 30fps, H.264 mp4, 비트레이트 4~6 Mbps**. 슬라이드에 박을 땐 무음 또는 BGM만.

## 공통 촬영 방법
- **권장 (PC)**: `scrcpy` + OBS 화면 녹화 (USB 디버깅 ON)
  ```
  scrcpy --max-size=1080 --video-bit-rate=8M --record=demo_v1.mp4
  ```
- **백업 (폰)**: Android 내장 화면 기록기 (상단 토글)
- **자막/캡션**: Shotcut 또는 CapCut 무료
- **압축**: 발표 PPT 안에 넣을 땐 `ffmpeg -i in.mp4 -vcodec libx264 -crf 26 out.mp4` 로 5MB 이하 목표

## V1: 보호자 회원가입 → 로그인 (30초)
- **목적**: 신규 사용자도 즉시 시작 가능함을 보여줌
- **장면**: 회원가입 → 자동 로그인 → 메인 도착
- **캡션 포인트**: "이메일·전화 검증 (3초)", "역할 자동 저장 (8초)"
- **주의**: 실제 ID/PW 노출 금지 — 데모 계정만 사용

## V2: 일정 추가 (AI 단계 생성 → 검토 → 저장) — 위험도 ★★★ (90초)
- **목적**: 이번 학기 핵심 변경(AI 검토 워크플로우) 강조
- **장면**:
  1. 일정 추가 화면 진입 (0~15s)
  2. 과업 선택(손씻기)·장소·특이사항 입력 (15~40s)
  3. "AI 단계 생성" 탭 → 로딩 → 단계 리스트 등장 (40~60s)
  4. 한 단계 직접 수정 (60~75s)
  5. 저장 → 일정 목록에 표시 (75~90s)
- **캡션 포인트**: "Gemini 2.5 Flash 호출 (45s)", "보호자가 수정 가능 (65s)", "DB CLOB 저장 (80s)"
- **주의**: 이 기능은 네트워크/API 키 의존도가 높아 라이브 시연 가장 위험. **반드시 녹화 필수**.

## V3: 사용자 앱 일정 알림 → 시작 → 단계 진행 → 완료 (120초)
- **목적**: 실제 사용자가 받는 경험 전달
- **장면**:
  1. FCM 푸시 도착 (0~10s)
  2. 알림 탭 → 일정 시작 화면 (10~20s)
  3. 단계 1~3 진행 (음성 안내 들리도록 폰 스피커 ON) (20~80s)
  4. 사진 인식이 들어가는 단계 (있으면) (80~100s)
  5. 마지막 단계 완료 → 칭찬 화면 (100~120s)
- **캡션 포인트**: "DB에 저장된 검토된 단계만 사용", "음성은 Gemini가 안내문구만 생성"
- **주의**: 폰 무음모드 해제, BGM 없이 원음 — 음성 안내가 들려야 함

## V4: AI 채팅 (똘똘이 대화) (60초)
- **목적**: chat 모드도 정상 동작
- **장면**: 사용자 앱 채팅 진입 → 음성 질문 2번 → 답변 2번
- **캡션 포인트**: "STT → Gemini → TTS"
- **주의**: 답변이 길어질 수 있으니 컷 편집

## V5: 요약 기능 (일정 완료 후) (30초)
- **목적**: 보호자가 한눈에 결과를 본다
- **장면**: 보호자 앱 일정 상세 → "요약 보기" → 한 문장 요약 등장
- **캡션 포인트**: "Gemini summarize endpoint"

## 녹화 우선순위 (시간 부족 시)
1. **V2** (이번 학기 핵심, 실패율 가장 높음)
2. **V3** (사용자 측 흐름 전체)
3. **V4 / V5** (보조)
4. **V1** (없어도 무방, 슬라이드 스샷으로 대체 가능)

## 발표 당일 체크
- [ ] PPT 안 비디오 자동재생 OFF (수동 클릭으로)
- [ ] 노트북 음량 70% 이상, 외부 스피커 확인
- [ ] 비디오 백업본 USB 1개 추가 보관
- [ ] 인터넷 끊겨도 PPT 안의 영상은 재생되는지 사전 테스트
