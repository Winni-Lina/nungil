# 눈길(Nungil) 시연 대본

발표일: 2026-06-08
분량: 5~7분
원칙: **안정 우선** — 신규 AI 기능은 코드/슬라이드로만 설명, 시연은 안정 빌드로만.

---

## 0. 시연 환경 체크리스트 (시연 1시간 전까지)

- [ ] 서버 로컬 실행 확인 (`mvn -pl server tomcat run` 또는 IDE 구동)
- [ ] Oracle DB 연결 확인 (`SELECT 1 FROM DUAL`)
- [ ] **마이그레이션 실행 확인**: `migration_custom_steps.sql` 적용 여부 — **적용 안 됐으면 신규 빌드 사용 금지** (사용자 앱이 custom_steps 컬럼을 읽다 NPE 가능)
- [ ] Gemini API 키 환경변수/properties 설정 (`google.ai.api-key`)
- [ ] FCM 서비스 계정 JSON 위치 확인
- [ ] 폰 2대 충전 80% 이상 + 동일 Wi-Fi 접속
- [ ] 폰 무음 해제, 음량 80%, 화면 회전 잠금 해제
- [ ] 폰 1 (보호자), 폰 2 (사용자) 앱 최신 안정 빌드 설치 확인
- [ ] 시연용 더미 계정 (`demo/1234`) 미리 가입 + QR 사용자 1명 등록
- [ ] 시연용 일정 1건 (예: "빨래하기" 14:00) 미리 등록
- [ ] 화면 미러링 도구 (scrcpy 등) 연결 테스트
- [ ] **마지막 빌드 동결 확정** — 발표 30분 전 이후 빌드 변경 금지

## 1. 도입 (40초)

**멘트**:
> "안녕하세요. 저희 팀은 중등도 지적장애인이 일상 과업을 스스로 수행할 수 있도록 돕는 듀얼 앱 '눈길'을 개발했습니다. 보호자가 일정을 등록하면, 사용자는 AI 안내를 받으며 단계별로 따라합니다."

**화면**: 슬라이드 1 (제목/팀 소개)

---

## 2. 아키텍처 한 장 설명 (50초)

**멘트**:
> "이번 학기에 AI 구조를 전면 재설계했습니다. 기존에는 실행 시점에 AI가 모든 판단을 했지만, 단계를 건너뛰는 버그가 있었습니다. 새 구조는 **(1) 보호자가 일정을 등록할 때 AI가 맞춤 단계를 생성하고, (2) 보호자가 검토·수정 후 저장하며, (3) 실행 시점에는 AI가 안내와 대화만 담당**합니다. Gemini의 system/user 프롬프트도 분리해서 응답 안정성을 높였습니다."

**화면**: 슬라이드 2 (아키텍처 다이어그램 - 기존 vs 신규)

## 3. 안정 기능 시연 - 보호자 (90초)

**멘트 + 동작**:
1. (폰 1) "보호자 앱입니다. `demo/1234`로 로그인합니다." → 로그인
2. "사용자 한 명을 미리 연동해 두었습니다." → 사용자 목록
3. "일정을 등록합니다. 빨래하기, 오후 두 시." → 일정 추가 화면 → 저장
4. "시간을 길게 눌러 드래그하면 즉시 변경됩니다." → 14:00 → 14:30 드래그 시연

**화면**: 폰 1 미러링

**비상 대응**:
- 서버 응답 지연 → "네트워크 점검 중인 듯합니다, 캐시된 화면으로 설명드리겠습니다" + 스크린샷
- 로그인 실패 → 미리 캡처한 진행 화면 슬라이드로 대체

## 4. 안정 기능 시연 - 사용자 (120초)

**멘트 + 동작**:
1. (폰 2) "사용자 앱입니다. 푸시 알림을 받으면 일정 화면이 열립니다." → 알림 탭
2. "AI가 음성으로 첫 단계를 안내합니다." → "빨래 모으기" 음성 들음
3. "사용자가 질문을 하면 AI가 도와줍니다." → "세제 어디 있어?" 발화
4. AI 답변 시연
5. "다 하면 말해줘" → "다 했어" 발화 → 다음 단계 진행

**화면**: 폰 2 미러링

**비상 대응**:
- 음성 인식 실패 → 텍스트 입력 fallback 사용
- Gemini 타임아웃 → "오프라인 데모 영상으로 보여드리겠습니다" → 사전 녹화 영상

## 예상 질문 + 답변 (Q&A 준비)

**Q1. AI 응답이 일관되지 않으면 어떻게 하나요?**
> system_instruction에 JSON 형식을 강제하고, 클라이언트에서 파싱 실패 시 fallback 메시지를 사용합니다. 또한 단계 이동 판단을 AI가 아닌 클라이언트 키워드(`다 했어`)로 처리해 신뢰성을 높였습니다.

**Q2. Gemini API 비용은?**
> 1회 호출당 토큰 수가 system 분리 후 평균 20% 감소했습니다. 일정 등록 시 1회 + 단계 평균 7회 호출 기준 사용자당 일 100원 미만으로 예상합니다.

**Q3. 지적장애인 사용자 검증은?**
> 이번 학기는 기능 구현에 집중했고, 실사용자 검증은 다음 단계입니다. 보호자 검토 UI를 통해 안전장치를 마련했습니다.

**Q4. 오프라인 동작은?**
> 음성 인식의 웨이크워드(Vosk)만 온디바이스이고, STT와 Gemini는 온라인 필수입니다. 향후 단계 안내 캐시를 통해 일부 오프라인 지원 예정입니다.

**Q5. 단계 생성이 부적절한 경우는?**
> 그래서 보호자 검토 단계를 둔 것입니다. 보호자가 저장 전에 단계를 자유롭게 편집/삭제/추가할 수 있습니다.
