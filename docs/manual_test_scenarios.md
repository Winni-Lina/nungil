작성일: 2026-06-07 (발표 전일)
대상 빌드: 동결 예정 빌드 (마이그레이션 `migration_custom_steps.sql` 적용 가정)

## TC-001: 보호자 회원가입 (정상)
**전제조건**: 앱 첫 실행, GUARDIAN 테이블 비어있음
**스텝**:
  1. 보호자 앱 실행 → 로그인 화면 → "회원가입"
  2. ID `test1`, PW `1234`, 이메일 `a@a.com`, 이름 `테스터`, 전화 `010-1111-2222` 입력
  3. ID 중복확인 버튼 → "사용 가능" 메시지
  4. 가입 버튼
**기대결과**: "가입 성공" 토스트, 로그인 화면 이동
**확인사항**: DB `GUARDIAN` 테이블에 row 1개
**위험도**: 낮음 (검증 완료)

## TC-011: 일정 추가 (AI 단계 자동 생성)
**전제조건**: 사용자 연동 완료, Gemini API 키 정상
**스텝**:
  1. 보호자 앱 → 일정 추가
  2. 과업 "빨래하기", 장소 "집", 일정 특이사항 "수건만 빨기"
  3. "AI 단계 생성" 버튼 → 단계 목록 자동 채워짐
  4. 확인 → 일정 저장
**기대결과**: 일정 저장, custom_steps 컬럼에 JSON 배열 저장
**확인사항**: Gemini 호출 1회, 단계 5~10개 생성됨
**위험도**: 높음 (신규 기능, 실기기 미검증)
