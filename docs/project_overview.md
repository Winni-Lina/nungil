# 눈길(Nungil) - 지적장애인 생활보조 AI

## 프로젝트 목표
중등도 지적장애인이 일상 과업(빨래, 설거지, 외출 등)을 보호자 도움 없이 수행할 수 있도록 AI가 음성으로 단계별 안내를 제공하는 듀얼 모바일 앱. 보호자는 일정과 단계를 사전에 검토·등록하고, 사용자는 음성 대화만으로 과업을 완수한다.

## 시스템 구성
- **보호자 앱 (Android)**: 일정/과업 등록, AI가 생성한 단계 검토, 사용자 진행 모니터링, 일정 완료 요약 확인
- **사용자 앱 (Android)**: 웨이크워드 인식, 음성 대화, 단계별 안내, 사진 인식, 화이트리스트 기반 앱 잠금
- **서버 (Spring Boot + Oracle)**: 일정/사용자/대화 관리, Gemini API 연동, FCM 푸시, STT

## 이번 학기 기여
1. **UI/UX 버그 9건 수정**: 회원가입 가독성, 일정 시간 변경 드래그, QR 모드, 세션 유지, 로그아웃 시 역할 유지 등
2. **AI 아키텍처 전면 재설계**
   - 기존: 실행 시점에 AI가 모든 판단 → 단계 건너뛰기, stepComplete 오판 등 버그
   - 신규: **생성 시점 맞춤 단계 생성 + 보호자 검토** → 실행 시점은 안내·대화 보조만 담당
3. **Gemini system/user 프롬프트 분리** (교수님 피드백 반영): 역할/규칙은 system_instruction으로, 일정·대화 데이터는 user prompt로 분리하여 응답 안정성 향상
4. **일정 완료 요약 기능 추가**: 일정 종료 시 질문 수·특이사항을 보호자가 확인 가능

## 시연 범위 (내일)
- **시연**: 회원가입/로그인, 사용자 QR 연동, 일정 등록/시간 변경, 사용자 앱 일정 실행(기존 안정 빌드), AI 채팅
- **코드·아키텍처 설명**: AI 단계 자동 생성, 보호자 검토 UI, 일정 완료 요약 (실기기 위험 회피 차원에서 시연 제외)

## 기술 스택
- **Backend**: Spring Boot 3 (Spring 6.1), MyBatis 3.5, Oracle, Google Gemini 2.5 Flash, Google STT, Firebase FCM
- **Frontend**: Android (Kotlin), Vosk(웨이크워드), Material 3, ZXing(QR), Retrofit/OkHttp
- **Test**: JUnit 5
