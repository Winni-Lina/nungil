## 2. 문제 정의
- 지적장애인은 단순해 보이는 일상 과업(손씻기, 빨래)도 단계가 흐리면 중도 포기
- 기존 보조 서비스는 보호자 중심이거나, AI 챗봇처럼 너무 일반적
- **핵심 문제**: 사용자별 인지 수준 차이를 반영한 "맞춤 단계"가 부재
- **멘트**:
  > "통계상 일상 과업 실패의 60% 이상이 다음 동작을 모르는 데서 발생합니다. 기존 앱은 정해진 매뉴얼만 보여줘서 개인차를 못 살립니다."
- **표시**: 페르소나 카드, 실패 시나리오 도식

---

## 3. 솔루션 개요
- **듀얼 앱**: 보호자 앱(등록·검토·모니터링) + 사용자 앱(음성·단계 안내)
- **AI**: Gemini 2.5 Flash로 보호자 입력→맞춤 단계 자동 생성
- **검토 워크플로우**: 보호자가 AI 결과를 수정해 저장 → 안전성 확보
- **멘트**:
  > "AI가 만든 안을 보호자가 한 번 거쳐 저장합니다. 실행 시점엔 검증된 단계만 사용자에게 갑니다."
- **표시**: 사용자/보호자 두 화면 나란히

## 6. AI 아키텍처 비교 (구 vs 신)
- `docs/architecture.md` 의 (B) sequence diagram 임베드
- 구: 매 단계 AI 호출(불안정/느림/통제불가) / 신: 등록 시 1회 생성·검토·저장 → 실행 시 안내만
- **멘트**:
  > "이전엔 사용자가 단계를 진행할 때마다 AI에 묻느라 느렸습니다. 지금은 보호자가 미리 검토해 저장하니, 사용자 앱은 빠르고 일관됩니다."
- **표시**: architecture.md (B) Mermaid, 호출 횟수 비교 표
